基于vue-cli搭建vue+vuex+axios+element-ui项目

如何基于vue脚手架搭建一个前端项目，其实是很容易的。本编文章就教大家如何基于vue的的脚手架集成vue+vuex+axios+element-ui的前端项目。那话就不多了，直接上干货。

快速开始

安装vue-cli

[vue init webpack projectName], 如果没有安装nodejs的，请先安装nodejs。然后根据对应的提示输入y/n,如图：


执行npm install 下载package.json中依赖的包，包安装完成后执行npm run dev，本地运行。在浏览器打开默认的localhost:8080，就可以看到对应的效果了。

接下来我们可以来看下项目结构，可以看到生成的项目结构如下图。

  • [build] webpack打包的相关配置，里面分成了dev和production两种打包模式，根据不同的环境进行打包。
  • [config] webpack打包不同环境的配置，这里是基本配置，我的做法是在项目根目录下配置一个.env.json文件，然后在这里引入，然后进行对应配置，不同的环境就改变env.json文件的参数。
  • [node_modules] 项目依赖的第三方库
  • [src] 重点目录结构，src下面我们可以到有:assets(存放一些图片)、components(公共组件)、router(vue-router配置)三个文件夹。
  • [static] 静态资源文件，在项目打包的时候不会被编译

其他的文件就不一一介绍了，可以自己去了解。在项目中也是很重要的，建议去了解一下。由于篇幅太长，觉得把vuex, router, axios，单独的分出来讲解。
