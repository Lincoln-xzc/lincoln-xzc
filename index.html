vuex

安装

npm install vuex es-promise –save这个命令来安装vuex和es6-promise，因为vuex依赖Promise。在src文件下面新建store文件，然后在store文件夹下面创建如下文件:

  • types.js 我们将常量放在types里面进行统一管理，方便协作,然后使用常量来代替事件类型。

  • mutation.js, 在vuex中改变state的方法就是提交mutation，Vuex 中的 mutation 非常类似于事件：每个 mutation 都有一个字符串的 事件类型 (type) 和 一个 回调函数 (handler)。

在组件里面可以使用this.$store.commit(‘xxx’)或者配合mapMutations来进行状态的改变，具体的用法自己看官网，官网介绍的很详细。

  • action.js，action.js里面是vuex state 异步的相关操作。你也可以在对应的action里面进行一些接口的调用，然后进行数据的存在，但是我个人不是很喜欢。

  • getters.js，getters.js是对state的数据进行一些操作，然后被返回。

在组件computed中通过this.$store.getters.doneTodos或者搭配mapGetters来使用。

  • store.js， 初始化state的值，并创建一个store。

配置好store以后，打开main.js,加入

Vuex在组件中管理数据是很方便的，在很多场景下都能使用，比如在列表中，记录查询条件，查询页数，然后进入详情页操作后，返回的时候从state里面取出对应的数据，进行从新渲染。虽然keep-alive也能实现如此的功能，但是keep-alive缓存了组件，不让组件从走生命周期，如果数据更新，列表数据却不进行更改。

项目中如果组件的传递数据不是很复杂，可以直接用EventBus的方式，不需要用到vuex,因为没有什么数据传递,存储，用vuex反而是个累赘。

基于vue-cli搭建vue+vuex+axios+element-ui项目

如何基于vue脚手架搭建一个前端项目，其实是很容易的。本编文章就教大家如何基于vue的的脚手架集成vue+vuex+axios+element-ui的前端项目。那话就不多了，直接上干货。

快速开始

安装vue-cli

[vue init webpack projectName], 如果没有安装nodejs的，请先安装nodejs。然后根据对应的提示输入y/n,如图：


执行npm install 下载package.json中依赖的包，包安装完成后执行npm run dev，本地运行。在浏览器打开默认的localhost:8080，就可以看到对应的效果了。

接下来我们可以来看下项目结构，可以看到生成的项目结构如下图。

  • [build] webpack打包的相关配置，里面分成了dev和production两种打包模式，根据不同的环境进行打包。
  • [config] webpack打包不同环境的配置，这里是基本配置，我的做法是在项目根目录下配置一个.env.json文件，然后在这里引入，然后进行对应配置，不同的环境就改变env.json文件的参数。
  • [node_modules] 项目依赖的第三方库
  • [src] 重点目录结构，src下面我们可以到有:assets(存放一些图片)、components(公共组件)、router(vue-router配置)三个文件夹。
  • [static] 静态资源文件，在项目打包的时候不会被编译

其他的文件就不一一介绍了，可以自己去了解。在项目中也是很重要的，建议去了解一下。由于篇幅太长，觉得把vuex, router, axios，单独的分出来讲解。

基于vue-cli构建的vue项目，优化webpack打包速度

wepack 打包vue项目速度优化

其实网上现在很多成熟的优化方案，本人这里也是做一个笔记，希望能给大家带来方便以及自己方便翻阅。
自己的项目是基于vue脚手架构建的一个项目，如果还不会构建vue项目，自己可以去网上查阅一下，很容易搞定。随着项目不断的增大，引入了很多第三方包，打包项目的时间也越来越久，终于忍受不了决定优化一下。可以看下现在打包所花费的时间，跟优化后的打包时间做下对比优化的话从以下几个方面进行

1.在webpack.base.conf.js中对vue-loader添加include,exclude.这样就能让vue-loader快速定位在那些目录文件需要处理。

  • include 表示哪些目录中的 .js 文件需要进行 babel-loader
  • exclude 表示哪些目录中的 .js 文件不要进行 babel-loader

在package.json文件中加入”dll:webpack -p –progress –config build/webpack.dll.conf.js”，然后运行npm run dll生成相应文件.

3.由于运行在 Node.js 之上的 Webpack 是单线程模型的，所以Webpack 需要处理的事情需要一件一件的做，不能多件事一起做。
我们需要Webpack 能同一时间处理多个任务，发挥多核 CPU 电脑的威力，HappyPack 就能让 Webpack 做到这点，它把任务分解给多个子进程去并发的执行，子进程处理完后再把结果发送给主进程。
